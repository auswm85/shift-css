## Overview

The `s-card` attribute provides a structured container for content with optional header, body, and footer sections.

## Basic Usage

```html
<article s-card>
  <div s-card-body>
    Simple card content
  </div>
</article>
```

## Full Structure

```html
<article s-card>
  <header s-card-header>
    <h3>Card Title</h3>
  </header>
  <div s-card-body>
    Main card content goes here.
  </div>
  <footer s-card-footer>
    <button s-btn="primary" s-size="sm">Action</button>
  </footer>
</article>
```

## Card Sections

| Attribute | Purpose |
|-----------|---------|
| `s-card` | Main card container |
| `s-card-header` | Card header area |
| `s-card-body` | Main content area |
| `s-card-footer` | Footer with actions |

## Examples

### Simple Card

```html
<article s-card>
  <div s-card-body>
    <h4>Simple Card</h4>
    <p>Just a body section with content.</p>
  </div>
</article>
```

### Card with All Sections

```html
<article s-card>
  <header s-card-header>
    <h3>Project Update</h3>
  </header>
  <div s-card-body>
    <p>The project is progressing well with all milestones on track.</p>
  </div>
  <footer s-card-footer>
    <button s-btn="ghost" s-size="sm">Cancel</button>
    <button s-btn="primary" s-size="sm">View Details</button>
  </footer>
</article>
```

## How It Works

Cards use CSS custom properties for consistent styling:

```css
:where([s-card]) {
  --_bg: var(--s-surface-raised);
  --_border: var(--s-border-default);
  --_radius: var(--s-radius-lg);
  --_padding: var(--s-space-4);

  background: var(--_bg);
  border: 1px solid var(--_border);
  border-radius: var(--_radius);
}

:where([s-card-body]) {
  padding: var(--_padding);
}
```

## Customization Examples

### Custom Background

```html
<article s-card style="--_bg: var(--s-primary-50);">
  <div s-card-body>
    Tinted card background
  </div>
</article>
```

### Sharp Corners

```css
[s-card] {
  --_radius: 0;
}
```
